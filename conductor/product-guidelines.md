# Product Guidelines

## 1. Prose Style

-   **Clarity and Conciseness:** All user-facing text, including agent responses, logs, and documentation, should be clear, concise, and easy to understand. Avoid jargon where possible.
-   **Professional but Approachable Tone:** The agent's personality should be helpful, professional, and slightly informal. It should guide users without being overly verbose.
-   **Active Voice:** Use the active voice to make instructions and explanations more direct (e.g., "The agent saved the file" instead of "The file was saved by the agent").

## 2. Branding

-   **Project Name:** The official name is `langunittest`. It should be written in lowercase unless at the beginning of a sentence.
-   **Consistency:** Maintain a consistent voice and style across all project materials, including the `README.md`, documentation, and agent interactions.

## 3. User Experience (UX) Principles

-   **Feedback is Key:** The agent must always provide clear feedback on its actions. Inform the user when a file is created, a test fails, or a command is successfully executed.
-   **Error Handling:** Errors should be handled gracefully. Provide informative error messages that suggest a solution or next step.
-   **Discoverability:** While the interface is a CLI, commands and options should be intuitive. Help text should be readily available.
-   **Non-Intrusive:** The agent should perform its tasks without requiring unnecessary user intervention. Default actions should be sensible and safe.
